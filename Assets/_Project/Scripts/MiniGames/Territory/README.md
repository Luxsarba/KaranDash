# Territory Paint MiniGame

Миниигра, где игрок закрашивает поле в жёлтый цвет, а 3 бота пытаются перекрасить его в свои цвета.

## Как работает игра

1. При старте сцены `TerritoryPaintPanel` генерирует сетку плиток (`gridWidth x gridHeight`).
2. Игрок красит плитку автоматически, когда находится над ней (без `E` и без отдельного режима ввода).
3. Каждый бот двигается по соседним клеткам (вверх/вниз/влево/вправо) и перекрашивает плитки в свой цвет.
4. Если игрок закрасил **последнюю плитку** конкретного бота:
   - бот останавливается на месте на `botDisableDuration` секунд (по умолчанию 10),
   - затем возвращается в игру и снова ходит, если игра ещё не завершена.
5. Победа наступает, когда **все плитки жёлтые**:
   - поле блокируется,
   - боты полностью останавливаются,
   - вызывается `onSuccess` (один раз).

## Скрипты

- `TerritoryPaintPanel.cs`  
  Главный контроллер: генерация поля, покраска, победа, управление ботами, гизмосы.

- `TerritoryPaintBot.cs`  
  Движение бота по клеткам, быстрая анимация шага, вызов покраски.

- `TerritoryPaintTile.cs`  
  Отдельная плитка: хранит владельца и меняет цвет через `MaterialPropertyBlock`.

## Основные настройки в инспекторе (`TerritoryPaintPanel`)

- `gridWidth`, `gridHeight` — размер поля.
- `tileSize`, `tileGap`, `tileThickness` — геометрия сетки.
- `boardOrigin`, `centerBoard` — позиционирование поля.
- `neutralColor`, `playerColor` — базовый цвет плитки и цвет игрока.
- `bots` — список ботов (цвет, базовая скорость, длительность анимации шага, визуал).
- `botSpeedMultiplier` — общий множитель скорости всех ботов.
- `botDisableDuration` — длительность временного отключения бота после потери всех его плиток.
- `onSuccess` — событие, которое вызывается при полном закрашивании поля игроком.

## Гизмосы

- `showBoardGizmos` — показывает границу всей игровой зоны.
- `showTileGizmos` — показывает сетку по клеткам.
- `boardGizmoColor`, `tileGizmoColor` — цвета гизмосов.

Это нужно для быстрой визуальной настройки зоны миниигры в Scene View.

## Важно

- Миниигра всегда активна, отдельный триггер запуска не нужен.
- `PlayerInteraction` не используется для этой миниигры.
- Управление игрока/пауза/курсор специально не перехватываются.
